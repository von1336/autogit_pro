# GitHub Automation Tool

Десктопное приложение для автоматизации работы с GitHub. Позволяет загружать файлы и папки в репозитории, управлять ветками, создавать Pull Request и репозитории через графический интерфейс.

## Возможности

- **Загрузка файлов и папок** — массовая загрузка с сохранением структуры директорий
- **Управление репозиториями** — просмотр, создание, удаление репозиториев
- **Управление ветками** — создание, переключение, удаление веток
- **Pull Request** — создание PR между ветками
- **Два метода загрузки** — через GitHub API или через Git (для больших файлов)
- **Сохранение учетных данных** — автоматический вход при повторном запуске

## Установка

### Вариант 1: Использование готового EXE

1. Скачайте `GitHubAutomation.exe` из папки `dist`
2. Запустите файл
3. Введите GitHub username и Personal Access Token

### Вариант 2: Запуск из исходного кода

1. Убедитесь, что Python установлен и добавлен в PATH

2. Установите зависимости:
```
install_dependencies.bat
```

Или вручную:
```bash
pip install -r requirements.txt
```

3. Запустите приложение:
```bash
python github_gui_ctk.py
```

### Вариант 3: Установка и сборка EXE

Запустите скрипт, который установит все зависимости и соберет EXE:
```
install_and_build.bat
```

## Получение GitHub Token

Personal Access Token необходим для доступа к GitHub API.

1. Откройте GitHub и перейдите в **Settings** (Настройки)
2. В левом меню выберите **Developer settings**
3. Перейдите в **Personal access tokens** -> **Tokens (classic)**
4. Нажмите **Generate new token (classic)**
5. Укажите название токена (например, "GitHub Automation")
6. Установите срок действия
7. Выберите разрешения (scopes):
   - `repo` — полный доступ к репозиториям
   - `delete_repo` — удаление репозиториев (если требуется)
8. Нажмите **Generate token**
9. Скопируйте токен и сохраните его (он показывается только один раз)

Формат токена: `ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

## Использование

### Вход в систему

При первом запуске введите:
- **Username** — ваш логин на GitHub
- **Token** — Personal Access Token

Данные сохраняются в файле `user_config.json` рядом с приложением.

### Загрузка файлов

1. Перейдите в раздел **Загрузка**
2. Нажмите **+ Добавить** для открытия файлового браузера
3. Выберите файлы и папки:
   - Обычный клик на папку — войти внутрь
   - Ctrl+клик — выбрать файл или папку
4. Выберите репозиторий из списка
5. Укажите ветку (по умолчанию `main`)
6. При необходимости укажите путь внутри репозитория
7. Нажмите **Загрузить на GitHub**

### Создание репозитория

1. Перейдите в раздел **Создать репозиторий**
2. Введите название репозитория
3. Добавьте описание (опционально)
4. Выберите тип: приватный или публичный
5. Выберите шаблон .gitignore
6. Нажмите **Создать**

### Управление ветками

1. Перейдите в раздел **Ветки**
2. Выберите репозиторий
3. Доступные действия:
   - Создание новой ветки
   - Удаление ветки
   - Просмотр списка веток

### Создание Pull Request

1. Перейдите в раздел **Pull Request**
2. Выберите репозиторий
3. Укажите исходную ветку (откуда)
4. Укажите целевую ветку (куда)
5. Введите заголовок и описание
6. Нажмите **Создать Pull Request**

## Сборка EXE

Для создания standalone EXE файла:

1. Убедитесь, что PyInstaller установлен:
```bash
pip install pyinstaller
```

2. Запустите сборку:
```
build_exe.bat
```

Или через Python:
```bash
python build_exe.py
```

EXE файл будет создан в папке `dist/GitHubAutomation.exe`

### Параметры сборки

- `--onefile` — один исполняемый файл
- `--windowed` — без консольного окна
- `--icon=icon.ico` — иконка приложения

## Методы загрузки файлов

Приложение поддерживает два метода загрузки:

### GitHub API (Contents API)

- Не требует установки Git
- Подходит для небольших файлов (до 100 MB)
- Каждый файл загружается отдельным запросом
- Медленнее для большого количества файлов

### Git (clone/push)

- Требует установленного Git
- Подходит для больших файлов и папок
- Быстрее для массовой загрузки
- Использует локальное клонирование репозитория

По умолчанию используется метод Git. Переключить метод можно чекбоксом в разделе загрузки.

## Решение проблем

### Ошибка "Token invalid" или "Bad credentials"

- Проверьте правильность токена
- Убедитесь, что токен не истек
- Проверьте, что токен имеет необходимые разрешения (scope `repo`)

### Ошибка "Repository not found"

- Проверьте название репозитория
- Убедитесь, что у токена есть доступ к репозиторию

### Ошибка при загрузке через Git

- Установите Git: https://git-scm.com/download/win
- Проверьте установку: `git --version`
- Настройте Git:
```bash
git config --global user.name "Ваше имя"
git config --global user.email "ваш@email.com"
```

### Данные не сохраняются после закрытия

- Убедитесь, что EXE файл не находится в защищенной папке
- Проверьте права на запись в папку с приложением
- Файл `user_config.json` должен создаваться рядом с EXE

### Приложение не запускается

- Установите Visual C++ Redistributable
- Запустите от имени администратора
- Проверьте антивирус (добавьте исключение)
